Security Architecture

Multi-Signature Verification

Cryptographic approval from multiple administrators before any link-containing announcement is published.

  • Configurable approval thresholds (2-of-3, 3-of-5, etc.)
  • Time-limited verification windows
  • Unique cryptographic codes per approver
  • Urgency is the red flag—friction is the feature

Hardware Wallet Binding

Cryptographic proof of admin identity. Even if a Discord account is compromised, attackers cannot approve without physical wallet access.

How It Works

The Verification Flow

From submission to publication. Every step verified, every action recorded.

1

Admin Submits Announcement

An administrator attempts to post a link-containing message to a protected channel. Validify intercepts and holds the content.

2

Approvers Notified

Designated approvers receive the message preview and a unique cryptographic verification code. Each code is single-use and time-limited.

3

Threshold Reached

Once the configured number of approvers submit their codes (e.g., 3 of 5), the verification threshold is satisfied. If the window expires, the message is rejected.

4

Published & Attested

The verified message is published with a Validify badge. Approval signatures are recorded on-chain, creating an immutable audit trail.

5

User Verifies in Browser

When a community member clicks the verified link, the Validify browser extension checks: Does this link match an attested announcement? Users see a clear trust badge before connecting their wallet.
